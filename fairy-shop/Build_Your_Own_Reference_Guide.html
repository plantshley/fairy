üóÇÔ∏è File Format Requirements

‚úÖ Recommended: SVG with Transparent Background

Why SVG?

  • Dynamic color filling via fill attribute
  • Infinite scalability without quality loss
  • Smaller file sizes than PNG for simple shapes
  • Easy to parse and manipulate paths

‚ö†Ô∏è PNG Alternative

If using PNG:

  • Must have transparent background (not solid white)
  • Color filling is limited - requires canvas pixel manipulation or color filters
  • Works best for textures and detailed illustrations that don't need recoloring
  • Larger file sizes at high resolution
üé® Color Filling Implementation: For dynamic color changes, SVG is strongly recommended. With SVG, you can change the fill property directly. With PNG, you'd need to use canvas filters (globalCompositeOperation) or WebGL shaders, which is significantly more complex.

File Preparation Checklist

  • Format: SVG files with transparent/no background fill
  • Paths: Ensure all shapes use <path> or <polygon> elements
  • Fill Attribute: Set default fill color or leave empty for dynamic filling
  • Viewbox: Include proper viewBox attribute for scaling
  • Organization: Group related paths in <g> elements
  • Naming: Use descriptive IDs for body parts (e.g., "body-bear", "limb-arm-left")

üì± Mobile Considerations

Touch Event Handling

React-Konva automatically handles touch events and maps them to mouse events. However, for optimal mobile experience:

  • touchstart: Equivalent to mousedown - initiates drag or draw
  • touchmove: Equivalent to mousemove - continues drag or draw
  • touchend: Equivalent to mouseup - completes action
  • Pinch gesture: Use Konva's built-in wheel event for zoom
  • Two-finger rotate: Enable with Transformer's rotation handles
üí° Mobile UX Tips:
  • Make touch targets at least 44x44px for comfortable tapping
  • Add visual feedback for drag operations (shadow, opacity change)
  • Implement long-press for additional options menu
  • Use larger Transformer handles on mobile (scaled by device pixel ratio)
  • Consider a floating toolbar instead of sidebar on small screens

üé® SVG vs PNG Color Filling

SVG Color Filling Recommended

// Load SVG as image const [image] = useImage('/path/to/shape.svg'); // Apply color with fill prop <Image image={image} fill={selectedColor} // ‚Üê Direct color change fillAfterStrokeEnabled={true} />

PNG Color Filling Limited

PNG color changes require filters or pixel manipulation:

// Method 1: Color filters (tinting only) <Image image={pngImage} filters={[Konva.Filters.RGB]} red={colorValues.r} green={colorValues.g} blue={colorValues.b} /> // Method 2: Canvas pixel manipulation (complex) // Requires iterating through pixels and replacing colors manually

Limitation: PNG color changes are approximations and don't work well for precise recoloring. Use PNG only for textures/details that don't need recoloring.
